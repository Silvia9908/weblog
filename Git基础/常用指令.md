### **一、仓库初始化与关联**

| 命令                                 | 作用                                                         |
| ------------------------------------ | ------------------------------------------------------------ |
| `git init`                           | 在当前文件夹初始化一个新的 Git 仓库（生成 `.git` 文件夹）    |
| `git clone <仓库地址>`               | 克隆远程仓库到本地（例如 `git clone git@github.com:用户名/仓库名.git`） |
| `git remote add origin <远程地址>`   | 将本地仓库关联到远程仓库（首次关联用）                       |
| `git remote -v`                      | 查看当前关联的远程仓库地址                                   |
| `git remote set-url origin <新地址>` | 修改关联的远程仓库地址（仓库改名后用）                       |

### **二、文件暂存与提交（本地操作）**

| 命令                       | 作用                                                         |
| -------------------------- | ------------------------------------------------------------ |
| `git status`               | 查看文件修改状态（哪些文件被修改 / 新增 / 删除）             |
| `git add <文件名>`         | 暂存指定文件（例如 `git add index.js`）                      |
| `git add .` / `git add -A` | 暂存所有修改（新增、修改、删除的文件）                       |
| `git commit -m "备注信息"` | 提交暂存区的文件到本地仓库（备注要清晰，例如 “修复登录按钮样式”） |
| `git commit --amend`       | 补充修改最近一次提交（避免多一条无用记录）                   |
| `git reset --hard HEAD~1`  | 撤销最近一次提交并删除修改（谨慎使用，会丢失内容）           |

### **三、远程同步（推送到 GitHub / 拉取更新）**

| 命令                          | 作用                                                        |
| ----------------------------- | ----------------------------------------------------------- |
| `git pull origin <分支名>`    | 拉取远程分支的最新内容到本地（例如 `git pull origin main`） |
| `git push origin <分支名>`    | 推送本地提交到远程分支（例如 `git push origin main`）       |
| `git push -u origin <分支名>` | 首次推送时绑定本地与远程分支，后续可简化为 `git push`       |
| `git push -f origin <分支名>` | 强制推送（覆盖远程仓库内容，仅自己开发时用）                |

### **四、分支操作**

| 命令                         | 作用                                                   |
| ---------------------------- | ------------------------------------------------------ |
| `git branch`                 | 查看本地所有分支（当前分支前有 `*`）                   |
| `git branch <新分支名>`      | 创建新分支（例如 `git branch dev`）                    |
| `git checkout <分支名>`      | 切换到指定分支（例如 `git checkout dev`）              |
| `git checkout -b <新分支名>` | 创建并切换到新分支（一步完成）                         |
| `git merge <分支名>`         | 将指定分支的内容合并到当前分支（例如 `git merge dev`） |
| `git branch -d <分支名>`     | 删除本地分支（已合并的分支）                           |

### **五、版本回溯与查看**

| 命令                       | 作用                                                         |
| -------------------------- | ------------------------------------------------------------ |
| `git log`                  | 查看提交历史（按 `q` 退出查看）                              |
| `git log --oneline`        | 简洁显示提交历史（一行一条）                                 |
| `git checkout <commit-id>` | 回溯到指定版本（`commit-id` 是 `git log` 中显示的哈希值前 6 位即可） |
| `git reflog`               | 查看所有操作记录（包括已删除的提交，用于找回误删版本）       |

### **六、撤销操作**

| 命令                       | 作用                                         |
| -------------------------- | -------------------------------------------- |
| `git checkout -- <文件名>` | 撤销工作区的修改（恢复到最近一次提交的状态） |
| `git reset HEAD <文件名>`  | 取消暂存（将已 `add` 的文件从暂存区移除）    |

### **常用场景组合**

1. **新文件 / 修改后推送到远程**：

   `git add -A` → `git commit -m "备注"` → `git push origin main`

2. **拉取远程最新代码**：

   `git pull origin main`

3. **创建分支开发新功能**：

   `git checkout -b feature/login` → 开发 → `git add -A` → `git commit -m "完成登录功能"` → `git push origin feature/login`

4. **撤销工作区修改**：

   `git checkout -- 文件名`
